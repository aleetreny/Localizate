# Documentación

La documentación se ha agrupado por dominio para que la raíz del repo se mantenga limpia y sea más fácil encontrar cada cosa.

- `docs/frontend/`: UX, contratos del frontend, análisis visuales y decisiones del mapa.
- `docs/modeling/`: ABT, validaciones, robustness, HPO y resultados de entrenamiento.
- `docs/data/`: inventarios de fuentes, snapshots, geografía y paneles socioeconómicos.
- `docs/project/`: estado operativo, bitácora y roadmap general.
- `docs/reference/`: glosarios y catálogos de variables usados por el producto.

El documento operativo principal sigue siendo `docs/project/STATUS.md`.
